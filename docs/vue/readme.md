## 总纲

## 单页面的优势

### 体验方面

每次刷新浏览器会有 CPU，内存的开销，进入网页消耗资源

- 流畅性
- 响应性

### 架构方面

- 前后端分离
- 商品展示从页面到组件级展示
- 数据驱动

### 生态方面

服务端渲染

### 性能安全方面

- 容错（组件出错，可以让别人看别的组件或者在组件内部处理）
- 缓存

## 缺点

- CPU 开销大
- 流量开销大（一次加载很多页面）
- 需要好的系统架构
- 搜索引擎不好（SEO）

## SSR

- 优点
  - 显著减少 TTI
  - SEO 解决方案
  - 内网数据组装
- 缺点
  - 服务器 CPU 压力大
  - CDN 不好支持
  - 开发工作量大
  - 全局对象失效（window）
